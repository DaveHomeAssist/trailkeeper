# VISUAL IDENTITY SYSTEM: "Trailkeeper"
**CORE AESTHETIC DIRECTION**

* Think:
  ```
  Backcountry ranger field journal meets GPS instrument panel.
  A worn USGS topo map folded too many times, crossed with the
  readout screen of a Garmin GPS unit mounted to a pack strap.
  Dark, precise, expedition-ready. Built for the trailhead, not the app store.
  ```
  
  ```
  19th-century cartographic expedition log reborn as local-first hiking software.
  Every element should feel like it was designed to survive rain, altitude, and
  three days without a charger.
  ```

* Key adjectives:
  ```
  Weatherproof. Field-ready. Deliberate. Expedition energy.
  Tactile precision. Backcountry authority.
  ```
  
  ```
  Not "outdoorsy app branding." Not REI catalog. Not Apple Fitness.
  More: USGS field survey equipment. Vintage expedition logbook.
  Instrument panel on a research vessel.
  ```

* Reference anchors:
  ```
  - USGS 7.5-minute topo quad maps (cream paper, contour lines, benchmark markers)
  - Vintage US Forest Service fire lookout station aesthetic
  - Garmin inReach / handheld GPS unit UI density
  - Edward Tufte's data visualization philosophy applied to trail data
  - John Muir's actual field journals — ink sketches, margin notes, bearing readings
  - WW2-era military field manuals (typography, stamps, grid overlays)
  ```

---

**TYPOGRAPHY FEELING**

* Primary display:
  ```
  Roboto Slab or similar sturdy slab serif — "field guide energy."
  Heavy enough to read on a muddy screen. Authoritative without being decorative.
  Think: the heading typeface of a wilderness first aid manual.
  Letter-spacing: slightly open. Weight: 700 for titles, 400 for body slab.
  ```

* Secondary UI:
  ```
  JetBrains Mono or similar monospace — "instrument panel."
  Every number (miles, elevation, bearing, count) should feel intentional and precise.
  The monospace makes data feel like it was measured, not estimated.
  Small caps for labels. Tight letter-spacing for coordinates.
  ```

* Notes:
  ```
  Never use rounded sans-serifs (no Inter, no SF Pro, no Nunito).
  The combination of slab serif + monospace creates the exact tension between
  "human field log" and "machine readout" that defines Trailkeeper's personality.
  Numbers especially: always monospace, always tabular figures if available.
  ```

## [IMAGE 3 — LOGO / SYMBOLIC MARK]

* Mark concept:
  ```
  A hexagonal compass rose where the directional spurs are replaced by
  stylized mountain peak silhouettes (three peaks: left small, center tall, right medium —
  the classic expedition peak grouping). The hex border has thin concentric topo
  contour rings inside it, like a benchmark on a quad map.
  At the center: the teal waypoint dot (#00E5C3) with a 1px circular reticle ring.
  Below the hex: the key blade from a compass needle pointing straight down,
  made of progressively thinning horizontal lines (same logic as Promptkeeper's
  key blade — but these are elevation contour bands, not DB rows).
  ```

* Geometry rules:
  ```
  Hex is the primary shape — six equal sides, flat-top orientation (not pointy-top).
  Interior topo rings: 3 concentric, each 4px inside the previous, 0.5px stroke.
  Mountain peaks: clean triangular silhouettes, no detail, apex touches inner topo ring.
  All strokes: 0.5–1.5px. No fills except the teal dot center.
  The mark must read clearly at 16px (favicon) — simplify ruthlessly below that size.
  ```

* Wordmark:
  ```
  TRAILKEEPER — Roboto Slab Bold, all caps, letter-spacing: 0.2em.
  Sized to match the hex width exactly (optically balanced).
  Color: aged parchment (#C8C2A0) on dark, deep ink on light.
  ```

* Tagline:
  ```
  BACKCOUNTRY FIELD LOG
  JetBrains Mono, 8–9px, letter-spacing: 0.18em, all caps.
  Color: muted slate (#4A5240) — secondary, never competing with the wordmark.
  ```

* Color rules:
  ```
  Mark on dark backgrounds: parchment (#C8C2A0) strokes, teal (#00E5C3) center dot only.
  Mark on light backgrounds: deep ink (#1A1C14) strokes, teal center dot.
  Never full-color fill on the hex body — always outline only.
  The teal dot is the ONLY accent color in the mark. Amber never appears in the logo.
  ```

* Texture rules:
  ```
  Subtle ink grain overlay at low opacity (5–8%) on the strokes — not on fills.
  This keeps the mark feeling hand-drawn at large sizes while staying clean at small.
  No gradients anywhere in the mark. Flat, precise, reproducible.
  ```

* Scalability notes:
  ```
  Favicon (16px): hex outline only + center teal dot. Drop wordmark, drop topo rings, drop peaks.
  Small (32–64px): hex + peaks + center dot. Drop topo rings.
  Medium (64–128px): full mark minus tagline.
  Large (128px+): full lockup with wordmark and tagline.
  ```

## CONSISTENCY REQUIREMENTS ACROSS ALL THREE

* Color palette:
  ```
  LOCKED — do not deviate:
  Background: #1A1C14 (Void Ink)
  Cards/panels: #252B1C (Undergrowth)
  Primary text: #C8C2A0 (Aged Parchment)
  Interactive/active: #00E5C3 (Teal Waypoint)
  Warning/index: #E8A844 (Amber Marker)
  Terrain/success: #4A7C59 (Pine Canopy) / #6DB87F (Pine Lit)
  Danger/trail blaze: #C45C2A (Blood Orange Blaze)
  Weather/offline: #7A8FA6 (Storm Grey)
  Achievement: #D4B483 (Summit Gold)
  ```

* Art style:
  ```
  Ink illustration / woodblock print hybrid with digital precision.
  Cross-hatch shading for shadows and depth.
  Clean, deliberate linework — no loose sketchy strokes.
  The overall feeling: a technically precise scientific illustration
  that happens to be of something alive and in motion.
  ```

* Texture:
  ```
  Micro-grain ink texture overlay (5–10% opacity) on all rendered surfaces.
  Topo contour lines as background fill pattern wherever space allows.
  Stamp-style worn edges on any rectangular elements (journal cover, patches, signs).
  Cross-hatch in shadows, parallel lines for atmospheric depth (fog, canopy light).
  ```

* Recurring motifs:
  ```
  - Hexagonal compass / benchmark hex shape
  - Topo contour lines (always curved, never straight — these are terrain, not grid)
  - Waypoint dot with circular reticle ring (the teal LED eye signature)
  - Mountain peak silhouettes (three-peak grouping)
  - Trail blaze marks (simple horizontal rectangles, blood orange)
  - Field journal pages with margin annotations and contour sketches
  - Distance/elevation numbers in monospace tabular figures
  - LOGGED / PACKED / SYNCED / OFFLINE stamps (worn block letter style)
  ```

* Mood:
  ```
  Expedition-hardened. Weatherproof. Built for permanence.
  The confidence of someone who has read the terrain, filed the trip plan,
  checked the conditions, and is already three miles in.
  Serious without being grim. Precise without being cold.
  ```

* NO:
  ```
  - Cute, friendly, bubbly, pastel, "outdoorsy lifestyle brand" energy
  - Soft rounded sans-serifs (no Inter, no Nunito, no Poppins)
  - Green + brown "nature app" color clichés (no flat lime green, no tan-brown)
  - Generic mountain silhouette logos (must be the specific hex-compass mark)
  - Photo-realistic rendering — this is always illustration, never photography
  - Clean white backgrounds — always aged parchment or dark ink base
  - Any element that would look at home in an Apple Fitness or Strava screenshot
  ```

## Notes

```
This is the sister identity to Promptkeeper (Notion Database Agent).
Both share the same base palette logic and "serious archivist / field operator" personality.
The key differentiation: Promptkeeper is indoor / archive / database.
Trailkeeper is outdoor / expedition / instrument.

Promptkeeper's motifs: database rows, Notion page icons, filing systems, index cards.
Trailkeeper's motifs: topo contours, trail blazes, compass bearings, elevation profiles.

Same palette family. Same ink illustration art style. Different domain, different motifs.
They should feel like tools built by the same person for different environments —
one for the desk, one for the trail.

For Midjourney: append --ar 1:1 --style raw --v 6 to the avatar prompt.
For full body: --ar 2:3 --style raw --v 6
For logo: --ar 1:1 --style raw --v 6 --no photorealistic,photo,3d render
```
